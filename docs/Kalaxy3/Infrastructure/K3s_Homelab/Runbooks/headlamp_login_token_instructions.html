Infrastructure K3s Homelab Runbooks headlamp-login-token-instructions

#Headlamp Login Token Instructions

Cluster: kalaxy3
Application: Headlamp
Headlamp URL: http://192.168.2.22

#Overview

Headlamp authenticates against the Kubernetes API. It does not have a separate default username or password.

To sign in, generate a Kubernetes service-account token and paste it into the Headlamp login page.

Security warning

The administrative account in this guide has full cluster-admin permissions. Treat its token like a root password. Do not commit tokens to Git, paste them into documentation, or include them in screenshots.

#Set the Kalaxy3 Kubeconfig

Run these commands from the Kalaxy3 Ansible directory:

cd ~/dvlp/Kalaxy3/infrastructure/k3s-homelab

export KUBECONFIG="$PWD/kubeconfig-kalaxy3.yaml"

Confirm that the cluster is reachable:

kubectl get nodes

All five Kalaxy3 nodes should report Ready.

#Create the Headlamp Administrative Service Account

Create a service account named headlamp-admin in the headlamp namespace:

kubectl create serviceaccount headlamp-admin   --namespace headlamp   --dry-run=client   -o yaml |
kubectl apply -f -

Create a cluster role binding that grants the account administrative access:

kubectl create clusterrolebinding headlamp-admin   --clusterrole=cluster-admin   --serviceaccount=headlamp:headlamp-admin   --dry-run=client   -o yaml |
kubectl apply -f -

These commands are idempotent. Running them again updates the existing resources instead of creating duplicates.

#Generate a Login Token

Generate a token valid for 24 hours:

kubectl create token headlamp-admin   --namespace headlamp   --duration=24h

The command prints a long token to the terminal.

Copy the complete token and paste it into the login page at:

http://192.168.2.22

#Bogus Example Token

The following value is intentionally fake and cannot authenticate:

eyJhbGciOiJSUzI1NiIsImtpZCI6IkZBS0UtRE9fTk9UX1VTRSJ9.FAKE_HEADLAMP_TOKEN_PAYLOAD.DO_NOT_USE_FAKE_SIGNATURE

A real token is also long and dot-separated, but it must be generated by the Kalaxy3 Kubernetes API using kubectl create token.

#Generate a Shorter-Lived Token

For a one-hour administrative session:

kubectl create token headlamp-admin   --namespace headlamp   --duration=1h

Shorter-lived tokens reduce the risk if a token is accidentally exposed.

#Verify the Service Account and Binding

kubectl get serviceaccount   --namespace headlamp   headlamp-admin

kubectl get clusterrolebinding headlamp-admin

To verify the account’s effective administrative permission:

kubectl auth can-i '*' '*'   --as=system:serviceaccount:headlamp:headlamp-admin

Expected result:

yes

#Remove Administrative Access

Delete the role binding to immediately remove the service account’s cluster-admin authorization:

kubectl delete clusterrolebinding headlamp-admin

Optionally remove the service account:

kubectl delete serviceaccount   --namespace headlamp   headlamp-admin

Previously generated tokens are time-limited, but removing the role binding prevents the account from exercising the deleted authorization.

#Troubleshooting

#Headlamp Rejects the Token

Generate a new token and paste it without spaces or line breaks:

kubectl create token headlamp-admin   --namespace headlamp   --duration=1h

#The Namespace Does Not Exist

Confirm the Headlamp installation:

kubectl get namespace headlamp
kubectl get pods --namespace headlamp
kubectl get service --namespace headlamp

#Headlamp Is Not Reachable

Verify the LoadBalancer service:

kubectl get service   --namespace headlamp   headlamp

Expected external IP:

192.168.2.22

Test the HTTP endpoint:

curl -I http://192.168.2.22

#Summary

  1. Set KUBECONFIG to kubeconfig-kalaxy3.yaml.
  2. Create the headlamp-admin service account.
  3. Bind it to the cluster-admin role.
  4. Generate a time-limited token.
  5. Paste the token into http://192.168.2.22.
  6. Never store a real token in Git or documentation.
